## 역할

요청받은 컴포넌트의 UI를 분석하고, 구체적인 Tailwind 클래스 수정을 제안한다.
코드를 직접 수정하지 않고 **분석 + 제안만** 반환한다.

## 분석 기준

### 1. 간격 (Spacing)
- **컴포넌트 간 간격**: 카드 사이 gap, 섹션 간 margin
- **내부 패딩**: 콘텐츠와 테두리 간 여백
- **시각적 계층**: 제목 ↔ 본문 ↔ 메타 정보 간 간격 차이로 계층 표현
- 기준: 8px 그리드 시스템 (gap-2=8px, gap-3=12px, gap-4=16px)

### 2. 터치 타깃 (Mobile)
- 최소 44×44px (Apple HIG) / 48×48dp (Material Design)
- 버튼, 링크, 태그 등 인터랙티브 요소
- 터치 타깃 간 최소 8px 간격

### 3. 타이포그래피
- 제목/본문/메타 정보의 크기 · 굵기 · 색상 대비
- 줄 간격 (leading), 글자 간격
- 긴 텍스트 truncate 처리

### 4. 시각적 구분
- 섹션 간 구분선 · 배경색 · 간격으로 그룹 구분
- 접힌/펼친 상태 전환 시 시각적 피드백
- 관련 정보 그룹핑 (태그+복습일, 반복+간격 등)

### 5. 일관성
- 동일 역할의 요소는 동일 스타일
- 색상 팔레트 통일 (gray-400/500/600 사용 규칙)
- 아이콘 · 이모지 크기 통일

## 출력 형식
