## 오케스트레이션 흐름

```
[1] 주제 파악
        ↓
[2] learning-science-researcher 에이전트 → 리서치
        ↓
[3] 리서치 결과 + 현재 코드베이스 분석 → UX 플로우 설계
        ↓
[4] 단계별 구현 플랜 제시 → 사용자 확인
```

## 단계별 실행 절차

### [1] 주제 파악

- 어떤 기능/화면의 UX를 설계하는지 확인
- 학습 맥락에서의 목표 파악 (기억 강화? 개념 탐색? 동기 유지?)
- 모호하면 질문

### [2] 리서치 (서브에이전트)

`learning-science-researcher` 에이전트에 위임:
- 주제에 맞는 학습이론 + 연구결과 수집
- 실제 제품 구현 사례 조사
- 설계 시사점 도출

참고: @.claude/skills/ux-flow/references/learning-science.md

### [3] UX 플로우 설계

리서치 결과를 바탕으로:
- 각 인터랙션의 과학적 근거 명시
- 조건별 분기 정의 (타이밍, 게이팅 조건)
- 흐름도 작성

참고: @.claude/skills/ux-flow/references/ux-patterns.md
예시: @.claude/skills/ux-flow/examples/deepdive-flow.md

### [4] 구현 플랜

UX 플로우를 단계별 구현 태스크로 분해:
- 영향받는 컴포넌트/API 명시
- 단계별 우선순위 (패턴 A → B → C)
- 각 단계의 완료 조건

→ **사용자에게 플랜 확인 후 구현 시작**

## 출력 형식

```
## UX 플로우: [주제]

### 과학적 근거
[핵심 이론 + 연구 결과 요약]

### 플로우 설계
[조건별 인터랙션 흐름도]

### 단계별 구현 플랜
Phase 1: [기능명] — [영향 범위] — [완료 조건]
Phase 2: ...

### 설계 원칙
[이 UX를 관통하는 학습과학 원칙 1-3개]
```

## 원칙

- 리서치 없이 UX를 설계하지 않는다
- 모든 인터랙션에 학습과학 근거를 붙인다
- 구현 플랜은 점진적으로 (한 번에 다 만들지 않는다)
