# 학습과학 리서치 가이드

`learning-science-researcher` 에이전트가 조사할 때 참고하는 이론 목록과
각 이론이 어떤 UX 문제에 적용되는지 매핑한다.

## 이론 → UX 문제 매핑

### 기억 강화 계열

| 이론 | 핵심 주장 | UX 적용 |
|------|-----------|---------|
| **Spaced Repetition** | 망각 직전 복습이 가장 효과적 | 복습 일정, 간격 조절 |
| **Testing Effect** | 시험(인출)이 재학습보다 기억 강화에 효과적 | 카드 뒤집기 방식, 자기평가 |
| **Generation Effect** | 스스로 생성하면 기억이 강화됨 | 힌트 점진 공개, 빈칸 채우기 |

### 심화 학습 계열

| 이론 | 핵심 주장 | UX 적용 |
|------|-----------|---------|
| **Elaborative Interrogation** | "왜?"를 묻고 답하면 기존 지식과 연결됨 | deepdive 카드 진입 타이밍 |
| **Self-Explanation Effect** | 스스로 설명하면 전이 능력 향상 | 요약 쓰기, 설명 입력 기능 |
| **Desirable Difficulty** | 적당히 어려운 것이 장기 기억을 강화 | 난이도 게이팅, 힌트 지연 |

### 구성 설계 계열

| 이론 | 핵심 주장 | UX 적용 |
|------|-----------|---------|
| **Interleaving Effect** | 개념 섞기가 구분 능력을 강화 | 복습 세션 구성, 카드 순서 |
| **Cognitive Load Theory** | 작업 기억 한계를 고려해 UI를 설계해야 함 | 화면당 정보량, 단계 분리 |
| **Pretesting Effect** | 먼저 시도해보는 것이 이후 학습 흡수율을 높임 | 탐색 전 퀴즈, 예상 질문 |

## 조사 시 확인할 출처 유형

1. **메타분석** — Dunlosky et al. (2013) 수준의 신뢰도 높은 종합 연구
2. **원 논문** — Roediger & Karpicke, Kornell & Bjork 등 핵심 연구자
3. **제품 사례** — Anki, Duolingo, Brilliant, SuperMemo의 구현 방식
4. **엔지니어링 블로그** — 실제 A/B 테스트 결과가 있는 자료 우선

## 시사점 도출 기준

리서치 결과를 아래 조건으로 필터링하여 앱에 적용 가능한 것만 선별:

- **조건 명확성**: "언제" 적용하면 효과적인지 조건이 있어야 함
- **반대 사례 인식**: 역효과가 나는 조건도 함께 정리
- **측정 가능성**: 앱 내 데이터(interval, easeFactor, rating)로 조건을 구현할 수 있어야 함
